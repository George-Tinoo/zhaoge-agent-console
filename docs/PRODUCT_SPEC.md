## 1. 产品概述

### 1.1 产品定位

**朝歌 (Chaoge)** 是大王的个人 AI 操作系统前端，是一个可扩展的 Agent Console / 智能体操作台。

**核心目标**:
- 统一智能体入口：整合妲己、子牙、Codex 及未来更多 Agent
- 管理多会话、多任务、多 Agent 协作
- 可视化工具调用、文件修改、命令执行、任务进度
- 支持从普通聊天逐步扩展到真实工程执行
- 为未来插件、设备控制、知识库、自动化任务预留架构

**产品口号**: "随身 AI 伴侣，数字生命中枢"

### 1.2 目标用户

**核心用户**: 追求 AI 原生体验的个人用户
- 希望建立深度个人 AI 关系的用户
- 需要多 Agent 协作完成复杂任务的用户
- 重视传统文化与个人运势结合的用户
- 希望 AI 主动服务而非被动响应的用户

### 1.3 核心价值主张

| 传统 AI App | 朝歌 Agent Console |
|------------|-------------------|
| 单一对话窗口 | 多 Agent 协作网络 |
| 被动等待提问 | 主动推送 + 任务管理 |
| 信息散落 | 项目 + 知识双维度沉淀 |
| 功能固定 | 插件化 + 开放扩展 |
| Web2 界面 | AI 原生交互 + HTML5 动态渲染 |

## 3. 功能模块详述

### 3.1 账号与设备系统

#### 3.1.1 端脑云账号体系

**账号规则**:
- 端脑云是唯一主账号
- 支持设备离线使用（本地缓存）
- 一个账号可绑定多台设备，但**同时只能使用一台**
- 一个设备只能绑定一个账号（不可多账号共享）
- 可在多个绑定的算力源（设备）之间**热切换**

**权限管理**:
- 当前仅管理员权限
- 暂不开放分级权限

#### 3.1.2 设备连接与发现

**连接方式**（优先级从高到低）:

1. **扫码绑定** - 扫描设备二维码完成绑定
   - 流程: 设备显示二维码 → App 扫码 → 确认绑定 → 建立连接
   - 安全: 二维码含时效性 token

2. **蓝牙发现** - App 自动发现附近可配对设备
   - 流程: 开启蓝牙 → 扫描附近设备 → 列表展示 → 点击连接
   - 范围: 10米内

3. **局域网查找** - 同一 WiFi 下自动发现设备
   - 流程: 连接 WiFi → mDNS 广播发现 → 展示同网段设备 → 点击连接
   - 适用: 家庭/办公室场景

**设备类型支持**:

| 设备类型 | 本地算力 | 云端算力 | 存储空间 | 典型 Agent |
|---------|---------|---------|---------|-----------|
| 端脑云 | 云端 GPU | 支持 | 云端存储 | 云端 Agent |
| 龙虾派 | 边缘算力 | 可调云端 | 32-512GB | OpenClaw, Hermes |
| AI NPC | 本地 120B 模型 | 可调云端 | ~100TB | 本地大模型 Agent |

#### 3.1.3 设备状态监控

**实时显示**:
- 在线/离线状态（绿色/红色指示灯）
- 存储空间剩余（**快满时红色百分比警告**，阈值 85%）
- 实时负载情况（CPU/GPU/内存使用率）
- 设备运算状态 - 是否忙碌、队列长度、预估等待时间

**多设备热切换**:
- 保持多个绑定的设备同时在线
- 用户可随时选择使用哪个设备的算力源
- 无缝切换，不中断当前会话

**设备离线处理**:
- 当前连接设备离线时，**自动切换到云端算力兜底**
- 提示用户: "之前连接的设备已离线，功能受限"
- 明确告知当前使用的是云端算力

**存储空间预警**:
- 设备存储快满时，**Agent 通过对话框主动提醒**
- 例: "大王，我的存储快满了 (92%)，需要清理一下吗？"
- 设备状态栏显示红色百分比警告
- 提供一键清理或手动管理选项

### 3.2 模型与参数系统

#### 3.2.1 模型选择

**选择入口**:
- 聊天界面随时切换模型（下拉选择器）
- 用户设置中统一配置默认模型
- **AI 原生方式**: 直接在聊天中自然语言切换
  - 例: "换 GPT-5.5"、"用轻量级模型"、"换个便宜的模型"

**支持模型类型**:
- GPT-5.5 (gpt55-server)
- Kimi K2.5 (cephalon/kimi-k2.5)
- Claude Code
- Codex (OpenAI)
- 本地模型 (龙虾派/AI NPC 本地部署)

#### 3.2.2 参数配置

**配置原则**:
- **全部参数对用户暴露**，用户有权了解全部内容
- 提供默认配置，但用户可完全自定义

**可调参数**:
- `temperature` - 创造性 (0-2)
- `max_tokens` - 最大输出长度
- `top_p` - 核采样
- `frequency_penalty` - 频率惩罚
- `presence_penalty` - 存在惩罚
- `system_prompt` - 系统提示词
- 其他模型特定参数

结合数据生成专属个人引导:
- 用户已提供的个人信息
- 当天地理位置
- 当天黄历信息
- 当天年月日时的八字
- 八字匹配测算

展示内容:
- 吉凶方位

- 事业顺利度

- 点击后进入当天的主要功能界面
- **每日重置**: 每天凌晨 6:00 后重置当天信息

**③ 形象设计区**
- **用户形象**: 上传照片为自己设计 AI 形象

角色来源:

- 形象 + 性格 + 声音 都可自定义

**⑤ 新功能推送区**
- 版本更新、新功能以对话窗口形式介绍
- 包含: 新功能是什么、怎么用、教学演示
- 让用户与最新功能保持同步

### 3.4 聊天系统

#### 3.4.1 聊天框布局 (消息展示区域)

**基础布局**:
- 智能体回复在**左侧**（带头像）
- 用户消息在**右侧**
- 所有文字**可被选取和复制**

**代码渲染**:
- 智能体回复中包含可执行代码时，在消息下方**直接渲染 HTML5 效果**
- 聊天框支持 HTML5 内容展示和交互
- 例: Agent 生成一个图表，直接在气泡下方渲染显示

**语音播报**:
- AI 可设定对我说文字或发语音
- **默认发送文字**，用户可要求 AI 改成发语音
- 用户可随时和 AI 交流，让它切换回复方式

**时间记录**:
- 每条消息显示时间戳
- 可折叠显示日期分隔

**消息操作**:
- **长按对方消息**可进行定向回复（引用回复）
- 长按显示操作菜单: 复制、转发、收藏、删除

**多媒体处理**:

| 媒体类型 | 支持操作 |
|---------|---------|
| **图片** | 点击放大浏览、双指缩放、存储保存、分享 |
| **音频** | 点击播放、进度拖动、转文字查看 |
| **视频** | 点击播放、全屏放大、存储保存 |
| **文件** | 预览、下载、打开第三方应用 |

**导航功能**:
- **一键到底**: 快速回到底部最新消息
- **悬浮导航按钮**: 聊天内容过长时，右侧显示悬浮按钮
  - 上下滑动快速定位到聊天任意位置
  - 按钮可隐藏/呼出

**聊天记录管理**:
- 每个聊天**独立记录**
- **界面左上角可展开聊天标题列表**
- 切换标题自动切换到对应聊天记录
- 根据已记录的**上下文继续后续沟通**

**未来聊天模式** (预留):
1. 自己 ↔ 自己的 Agent
2. 自己 ↔ 人类好友
3. 自己 ↔ 别人的 Agent
4. 自己的 Agent ↔ 别人的 Agent

#### 3.4.2 输入框设计

**附件添加**:
- **手机本地**: 照片库选取、相机拍照
- **连接设备**: 可添加设备上的文件（照片、视频、PDF、压缩包等所有格式）

**输入方式**:
- **打字输入**: 常规键盘输入
- **语音输入**: **长按输入框**切换为语音输入状态
  - 长按状态下说话自动识别
  - 停止说话后自动转为文字填入输入框
  - 用户点击发送后才发送

**快捷功能按钮** (输入框右侧):

| 按钮 | 功能 |
|-----|------|
| **相机** | 唤起手机相机，实时拍照/录像，完成后直接发送 |
| **录音** | 唤起录音功能，实时录音，完成后发送录音文件 |
| **文件** | 选择本地或设备文件发送 |

**桌面端专属**:
- **文件拖拽上传**: 鼠标拖拽文件直接拖到对话框

### 3.5 多 Agent 协作系统

#### 3.5.1 Agent 基础配置

每个 Agent 配置三项基础设定:

```typescript
interface Agent {
  id: string;
  name: string;              // 显示名称
  avatar: string;            // 头像图片
  role: string;              // 角色定义（身份、性格）
  llmProvider: string;       // 大模型服务提供商
  llmModel: string;          // 具体模型
  capabilities: string[];    // 擅长的工作类型
  isSupervisor: boolean;     // 是否主管 Agent
}
```

**工作类型** (capabilities):
- 记录/记忆
- 数据分析
- 代码开发
- 绘画创作
- 文案写作
- 项目管理
- 更多可扩展...

**Agent 来源**:
- 云端镜像（OpenClaw、Hermes Agent 等）
- 本地设备（龙虾派、AI NPC 上安装的 Agent）
- 不同 Agent 可用相同或不同的 LLM 后端

#### 3.5.2 主管 Agent 机制

**角色定位**:
- 用户的**主要交流对象**
- 负责将用户需求拆解为不同类型的工作任务
- 组织协调不同 Agent 分工协作
- 汇总各 Agent 工作结果，形成完整结论给用户

**工作流程**:

```
用户需求 → 主管分析 → 拆解子任务 → 汇报方案 → 用户确认
                                            ↓
用户收到汇总 ← 结果整理 ← 各 Agent 执行 ← 分配任务
```

**先汇报后执行**:
1. 用户向主管 Agent 提出需求
2. 主管分析需求，拆解为子任务
3. **向用户汇报组织方案和计划**，等待确认
4. 用户确认后，分配给各 specialist Agent 执行
5. 收集结果，汇总整理，反馈给用户

**特殊情况**:
- 若只有一个 Agent，该 Agent 既是主管也是执行者
- 主管 Agent 也可直接处理具体事务

**示例配置**:

| Agent | 角色 | 职责 |
|-------|------|------|
| 妲己 | 主管 Agent | 与用户交流、任务拆解、协调分工、结果汇总 |
| 子牙 | Specialist | 复杂架构、代码开发、技术分析、方案评审 |
| Codex | Specialist | 多文件开发、测试、重构、生成 diff |

#### 3.5.3 直接唤起与独立对话

**唤起方式**:
- 用户直接叫 Agent 的名字（如"子牙，帮我看看这个代码"）
- 被叫到的 Agent 立即唤起，通过 App 独立聊天窗口与用户交流
- 每个 Agent 拥有**独立的聊天频道**

**使用场景**:
- 用户可直接与任一 Agent 单独交流
- 也可只和主管 Agent 交流，由主管协调其他人

#### 3.5.4 动态任务分配机制

**新任务类型处理流程**:

1. **发现问题**:
   - 用户提出的需求涉及未指定负责 Agent 的功能
   - 例: 需要绘画 Agent，但目前只有文案和代码 Agent
   - 主管 Agent 报告用户: "目前没有负责此功能的 Agent"

2. **内部讨论**:
   - 主管组织**所有现有 Agent 内部讨论**
   - 讨论内容:
     - 现有 Agent 谁更适合承接这个新任务？
     - 如果都不适合，需要创建什么样的新 Agent？
     - 新 Agent 应该用什么大模型/后端服务支持？

3. **用户决策**:
   - 讨论结束后，用户可要求主管:
     - 创建一个新的 Agent 专门负责此类任务
     - 或分配给某个现有 Agent 兼管

4. **后续执行**:
   - 新 Agent 创建/分配后，自动纳入主管的调度体系
   - 未来同类任务直接分配给该 Agent

### 3.6 文件与知识库系统

#### 3.6.1 双维度沉淀体系

内容以**两个维度**呈现在 App 中:

| 维度 | 项目 (Project) | 知识 (Knowledge) |
|------|---------------|-----------------|
| **分类方式** | 按任务/目标/事项分类 | 按主题/领域分类 |
| **内容来源** | 特定项目的沟通内容 | 跨项目的同类知识 |
| **结构** | 动态列表，Agent 自动归类 | 主题目录，动态增加 |
| **存储内容** | 聊天记录、文件索引、多媒体 | 领域知识、经验沉淀 |

#### 3.6.2 项目库 (Project Library)

**入口位置**: 聊天界面**右上角项目按钮**

**功能特性**:
- 所有沟通内容**自动分类规整**为项目列表
- 每个项目包含:
  - 项目相关所有聊天记录
  - 自动分类整理的内容
  - 文件索引、图片、视频、多媒体文件位置
- **动态增长**: Agent 自动识别新事项并创建/归类项目
- **无法归类的内容**: 统一放入"杂项"或等待 Agent 后续整理

**使用场景**:
- "装修项目" - 所有装修沟通、设计图、预算表
- "小说创作" - 章节讨论、角色设定、剧情大纲
- "代码开发" - 需求讨论、代码文件、测试记录

#### 3.6.3 知识库 (Knowledge Base)

**定义**: 按**主题/领域**组织的内容，可能跨多个项目

**功能特性**:
- 同方向、同领域的知识归入同一目录
- **动态增加**: Agent 识别新知识点自动扩展目录
- 支持多层级目录结构
- 存储文件索引和多媒体资源位置

**使用场景**:
- "AI 技术知识" - 跨项目的代码经验、架构思路
- "育儿知识" - 宝宝成长相关信息（跨聊天记录）
- "设计规范" - 装修、UI 等设计原则沉淀

#### 3.6.4 内容沉淀机制

**来源覆盖**:
- ✅ 用户与 Agent 的沟通
- ✅ 用户与其他用户的沟通
- ✅ Agent 与 Agent 之间的沟通
- ✅ 所有沟通中的文件、图片、视频、媒体

**存储方式**:
- 实际文件存储在连接的设备/云端（龙虾派、AI NPC、端脑云）
- App 中存储**文件索引**和**元数据**
- 多媒体文件位置统一记录，方便调用

#### 3.6.5 记忆共享机制

**核心能力**:
- 即使**换不同的 Agent 沟通**，也能访问之前沉淀的全部记忆
- 所有 Agent 共享项目和知识库的内容
- 有结构的内容沉淀支持长期工作和学习
- 新 Agent 加入时可快速了解历史背景

### 3.7 主动提醒与任务系统

#### 3.7.1 推送机制

**核心能力**:
- 突破当前 App **无推送**的限制
- **Agent 可主动给用户发消息**，而非被动等待用户提问
- 用户规划的任何事项，Agent 可在设定时间**主动提醒和反馈汇报**

**价值**:
- 提高用户互动频率和持续使用概率
- 任务执行过程中让用户明确了解进度
- 形成"Agent 在替我盯进度"的感知

#### 3.7.2 主动提醒内容

**可设定提醒的事项**:
- 定时任务提醒（会议、deadline、纪念日）
- 任务进度汇报（阶段性完成、需要决策、遇到阻碍）
- 每日信息推送（运势、日程概览、待办提醒）
- 异常/风险告警（设备离线、存储将满、任务延期）
- 用户自定义的任何希望 Agent 主动反馈的事项

**互动特性**:
- 用户可**删减、修改**已设定的提醒
- 用户可**新增**更多自定义提醒

#### 3.7.3 HTML5 动画欢迎效果

**呈现形式**:
- 利用聊天框**可渲染 HTML5** 的特性
- 设计**简单动画效果**（像素风格）
- 每次主动推送时呈现:
  - 说话者是谁（哪个 Agent）
  - 主题是什么
  - 友好的打招呼互动

**效果示例**:
- 妲己推送时: 像素风狐妖形象 + "大王，妲己来报~"
- 子牙推送时: 像素风智者形象 + "主公，有要事禀报"
- 任务提醒时: 像素风日历/时钟动画 + 任务标题

#### 3.7.4 任务系统架构

**任务记录维度**:
```typescript
interface Task {
  id: string;
  title: string;           // 任务标题
  description: string;     // 任务描述
  startTime: Date;         // 开始时间
  deadline: Date;          // 截止时间
  reminderTime: Date;      // 提醒时间
  location?: string;       // 地点
  participants: string[];  // 参与者
  assignee: string;        // 负责人
  agentId: string;         // 负责的 Agent
  status: TaskStatus;      // 状态
  goal: string;            // 预期成果
  collaborationStatus: string; // 协作状态
  risks: string[];         // 风险
```

### 3.8 跨端体验设计

#### 3.8.1 手机端 (随身终端 - 必备)

**核心定位**:
- 手机端是**随身终端**，AI 时代的主力入口
- **所有用户必须拥有移动端 App**
- 电脑端登录需通过移动端扫码（或账号密码备用）

**已覆盖功能**:
- 选中复制粘贴
- 上下滚动 + 快速导航
- 相机拍照/录像
- 录音功能
- 多媒体输入（本地 + 设备文件）

**核心输入方式**:
- **首选语音交流** - 语音直接输入或语音转文字
- 键盘打字为辅助

**文件存储**:
- 生成的文件默认存储到连接的云端/龙虾派/AI NPC
- 通过 HTML5 界面可直接预览部分文件

#### 3.8.2 桌面端 (生产力扩展)

**核心定位**:
- 独立的桌面应用程序
- 与手机端界面**高度一致**，仅调整为大屏比例
- 作为生产力场景的扩展，**非必需但增强体验**

**登录方式**:
1. **首选**: 移动端 App 扫码登录（保证一致性）
2. **备用**: 手机号 + 验证码/密码登录

**桌面端专属增强**:
- **文件拖拽上传**: 鼠标拖拽文件直接拖到对话框上传
- **本地文件存储**: 文件可直接保存到电脑本地（而非仅云端/设备）
- 更方便用户在本地对文件进行后续操作

**功能一致性**:
- 支持录音、摄像头、多媒体输入（电脑通常都有麦克风摄像头）
- 所有手机端功能桌面端都支持

**核心输入方式**:
- **首选键盘打字** - 桌面办公场景更高效
- 语音输入为辅助（可用电脑麦克风）

#### 3.8.3 输出方式 (两端通用)

**默认输出**:
- **文字为主**，语音为辅

**语音输出选项**:
- 用户可要求 Agent "用语音回复"

**对讲模式** (默认):
- 一条一条语音，需点击播放

**实时交流模式**:
- App 时刻听取外部声音
- 用户说话告一段落后自动回复
- 回复音频直接播放
- Agent 像参会者一样实时听到所有言论，实时表达观点

#### 3.8.4 跨端协同对比

| 特性 | 手机端 | 桌面端 |
|------|--------|--------|
| **定位** | 随身终端（必备） | 生产力扩展（增强） |
| **登录方式** | 主账号直接登录 | 扫码登录（首选）/ 账号密码 |
| **核心输入** | 语音为主 | 键盘为主 |
| **文件存储** | 云端/设备 | 本地 + 云端/设备 |
| **专属功能** | 随身便携 | 拖拽上传、本地存储 |
| **界面** | 移动端比例 | 桌面大屏比例 |
| **功能覆盖** | 完整功能 | 完整功能 + 桌面增强 |

### 3.9 权限与安全系统

#### 3.9.1 当前权限模型 (单用户模式)

**现状**:
- 当前设计为**单一主人**使用自己的 App
- 用户通过账号登录后，**可执行所有操作**
- 对自己的设备、文件、Agent 拥有完全控制权

#### 3.9.2 未来扩展: 跨实体通信权限

**适用场景**:
- 我的 Agent ↔ 其他人类用户
- 我的 Agent ↔ 其他用户的 Agent
- 涉及**对外信息传递**和**跨域动作执行**

**信息传递权限规则**:

**基本原则**:
- **任何信息传递给外部实体，必须得到用户审批同意**
- 无论是传给其他人类还是其他 Agent

**审批模式**:
- **每次审批**: 每次传递都需用户确认（最高安全）
- **批量审批**: 用户可确认"同类内容后续是否仍需审批"
  - 例: "是否允许向该用户持续分享项目进度，无需每次确认？"
  - 用户可随时撤销批量授权

**动作执行权限规则**:

**基本原则**:
- **每一次动作执行都需要得到用户认可**
- **无例外**（即使是看似无害的操作）

**执行前确认**:
- 动作内容预览
- 影响范围说明
- 用户明确点击"同意执行"后方可继续

#### 3.9.3 高危行为标记与提醒

**自动标记为高危的场景**:

1. **涉及个人隐私信息**:
   - 身份证号、住址、联系方式
   - 家庭成员信息
   - 财务账户信息
   - 健康医疗记录
   - 其他敏感个人数据

2. **涉及资金安全**:
   - 任何支付、转账、消费行为
   - 账户密码、支付密码相关
   - 投资、理财操作
   - 合同签署、资金授权

**高危行为处理**:
- **醒目标识**: 红色警告、特殊图标、明显提示
- **重点提醒**: 强制用户审查内容细节
- **二次确认**: 必须明确输入"确认"或生物识别验证
- **记录留痕**: 所有高危操作记录日志，便于追溯

#### 3.9.4 权限审批界面设计

**审批弹窗要素**:
- 谁在请求？（Agent 名称 + 头像）
- 请求类型？（信息传递 / 动作执行）
- 具体内容？（展示完整内容预览）
- 传递给谁？（目标人类/Agent 身份）
- 风险等级？（普通 / 高风险）
- 操作按钮？（同意 / 拒绝 / 修改后同意）

**批量授权管理**:
- 设置中可查看所有已授权的批量权限
- 可随时撤销特定授权
- 定期提醒用户审查现有权限

#### 3.9.5 安全原则总结

| 场景 | 权限要求 |
|------|---------|
| 用户自己的设备/文件 | 完全控制，无需额外确认 |
| 信息传递给外部 | **必须审批**，可选批量授权 |
| 动作执行（任何） | **必须审批**，无例外 |
| 涉及隐私/资金 | **高危标记**，重点提醒，二次确认 |
